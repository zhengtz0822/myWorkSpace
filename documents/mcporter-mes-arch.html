SKILL + Gateway 双重驱动
📋
SKILL 层（智能指导）
指导 Agent 正确调用
每个业务域一个 SKILL，告诉 Agent：

• 这个域能查什么（工具清单）
• 该调哪个 Server（mes-production / mes-quality ...）
• 参数怎么传（字段说明 + 示例）
• 边界在哪（跨域时转到对应 SKILL）

作用：防止 Agent 调错域、传错参，确保准确驱动
🛡️
Gateway 层（权限控制）
按用户 + 域控制工具可见性
MCP Gateway 为不同用户开放不同工具：

作用：即使 Agent 尝试调用，Gateway 也会拦截越权请求
业务层抽象
👤
接入层
Access Layer
职责：用户身份识别 + 消息路由

• 钉钉/飞书/企微 → 统一 inbound metadata
• 提取 userId userName deptId
• 不同渠道适配，业务层无感知
🧠
智能层
Intelligence Layer
职责：意图理解 + 域识别 + 参数提取

• 自然语言 → 识别业务域 → MCP 工具调用
• "查工单" → 生产域 → mes-production.get_workorder
• SKILL 指导：正确域 + 正确工具 + 正确参数
📦
协议层
Protocol Layer
职责：MCP 通信 + 身份透传 + 权限执行

• mcporter 动态配置注入用户身份
• X-User-Id / X-User-Name headers
• Gateway 按域路由 + 按权限过滤工具
⚙️
业务系统层
Business System Layer
职责：多域 MCP Server + 开放 API

• 一域一 Server，独立部署独立演进
• 每个域封装自己的查询工具
• 调用 MES 开放 API 获取实际数据
完整数据流（单次查询）
1
员工在钉钉发消息
"帮我查一下工单 WO-001 的进度"
2
钉钉机器人 → OpenClaw
消息 + inbound metadata（userId, userName, deptId）传入 OpenClaw
3
OpenClaw Agent 意图识别 + 域识别
NLU 解析 → 意图：查询工单 → 域：生产域 → 工具：mes-production.get_workorder → 参数：id=WO-001
3.5
SKILL 匹配 → 加载 mes-production SKILL
SKILL 指导 Agent：调用 mes-production Server，使用 get_workorder 工具，传入 id 参数
4
Agent 调用 mcporter-with-user.ps1
传入 -Tool "mes-production.get_workorder" -UserId "A001" -UserName "张三" -ToolArgs '--args \'{"id":"WO-001"}\'
5
脚本生成临时配置 + 调用 mcporter
读取模板 → 替换 {{USER_ID}}/{{USER_NAME}} → 写入临时 JSON → mcporter call --config 临时配置
6
mcporter → MCP Gateway
HTTP 请求携带 Authorization + X-User-Id + X-User-Name headers，目标 mes-production Server
7
MCP Gateway 鉴权 + 权限校验 + 路由
验证 Token → 检查该用户是否有权访问生产域 → 路由到 mes-production MCP Server
8
mes-production MCP Server → MES 开放 API
调用 MES 工单查询 API，返回结构化结果
9
结果逐层返回 → OpenClaw 生成自然语言回复
MES API → MCP Server → MCP Gateway → mcporter → Agent → "工单 WO-001 当前状态：生产中，完成度 75%"
10
钉钉员工收到回复
脚本自动清理临时配置文件，整个流程完成
